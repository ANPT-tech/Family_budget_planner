# Family Budget App

## Описание проекта

Мобильное приложение «Планировщик семейного бюджета» предназначено для учета доходов и расходов, анализа финансового состояния и совместного управления семейным бюджетом.

## Цель проекта

Создание удобного инструмента для контроля финансов, повышения финансовой грамотности пользователей и планирования семейного бюджета.

## Основной функционал

* регистрация и авторизация пользователей;
* создание семейного бюджета;
* добавление доходов и расходов;
* управление категориями;
* установка лимитов по категориям;
* просмотр отчетов и аналитики;
* получение уведомлений о превышении лимитов.

## Стек технологий

* Kotlin — мобильное приложение (Android);
* Python (Django) — серверная часть;
* PostgreSQL — база данных;
* Git, GitHub — система контроля версий;
* Android Studio — среда разработки.

## Архитектура системы

Приложение реализовано по клиент-серверной архитектуре:

* клиент (мобильное приложение) взаимодействует с сервером через API;
* сервер обрабатывает запросы и взаимодействует с базой данных;
* обмен данными осуществляется в формате JSON по протоколу HTTP/HTTPS.

## Структура проекта

Проект представляет собой мобильное Android-приложение и имеет следующую структуру:

* app/ — основной модуль мобильного приложения;

  * src/ — исходный код приложения;

    * main/ — основной код приложения;
    * test/ — модульные тесты;
    * androidTest/ — инструментальные тесты;
* gradle/ — файлы конфигурации системы сборки Gradle;
* build.gradle — основной файл сборки проекта;
* settings.gradle — настройки проекта;
* gradlew, gradlew.bat — скрипты для запуска Gradle;
* google-services.json — конфигурация сервисов Google (Firebase);
* proguard-rules.pro — настройки оптимизации и обфускации кода;
* README.md — документация разработчика.

Дополнительно в рамках проекта могут использоваться следующие директории:

* docs/ — документация проекта;
* diagrams/ — диаграммы (UML, DFD, Use Case);
* design/ — макеты пользовательского интерфейса.


## Запуск проекта (планируемый)

1. Установить необходимые зависимости.
2. Запустить серверную часть (Django).
3. Запустить мобильное приложение в Android Studio.
4. Подключиться к API сервера.
